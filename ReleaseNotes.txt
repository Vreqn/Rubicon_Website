2026-06-10 — Hero video live (Session 1, part 4)
=========================================================
- Krisztian's hero video wired in. Source "Rubicon Website Hero
  Final.mp4" (1920x1080, 27.4s, ~20 Mbps, 66.6 MB) was far too heavy for
  web. Re-encoded with ffmpeg: H.264 high/yuv420p, CRF 28, preset slow,
  AUDIO STRIPPED (hero is muted), +faststart, scaled to 1920 wide.
  Result: 2.39 MB @ 729 kbps — 96% smaller, quality ample behind the
  50%-opacity scrim. Poster frame (t=2s) at 140 KB.
- Files: public/video/hero.mp4 + public/video/hero-poster.jpg (matches
  the HeroVideo drop-in contract; auto-detected at build, no code change).
- 66 MB master kept at _assets-source/hero-master.mp4 (gitignored). To
  re-encode: ffmpeg -i _assets-source/hero-master.mp4 -an -c:v libx264
  -crf 28 -preset slow -movflags +faststart public/video/hero.mp4
- WebM/VP9 variant tried (CRF 36) but landed LARGER than the H.264
  (3.07 vs 2.39 MB) — dropped; ship MP4 only (universal support anyway).
- Verified in-browser via scripts/hero-video-check.mjs: readyState 4,

2026-06-10 — Initial scaffold (Session 1)
=========================================================
- Greenfield scaffold created manually (no `npm create astro` — the wizard
  targets Astro 6 templates and prompts on non-empty directories).
- Astro pinned to ^5.18.2, deliberately NOT v6: Astro 6 removed support for
  the `src/content/config.ts` location that project CLAUDE.md mandates.
  v5 supports that path plus the modern Content Layer API (glob loaders).
  The v6 upgrade later = move file to `src/content.config.ts` + amend
  CLAUDE.md. The v5 deprecation notice about the config location is known
  and accepted — do not "fix" it without amending CLAUDE.md first.
- Packages: astro ^5.18.2, @astrojs/sitemap ^3.7.3,
  @fontsource/space-grotesk ^5.2.10 (400/500),
